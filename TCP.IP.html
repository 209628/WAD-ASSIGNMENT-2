TRANSMISSION CONTROL PROTOCOL/INTERNET PROTOCOL

Definition:

- A layered networking framework that explains how data is communicated between devices over a network using standardized protocols.

TCP/IP Model layers:

  1. Application layer: It is the one that users interact with directly. It provides protocols that applications use to communicate over a network such as HTTP for web browsing, FTP for file transfer, and SMTP for email.
  2. Transport layer: is responsible for ensuring that data is delivered reliably and in the correct order between two devices. It breaks data into smaller packets before sending and reassembles them at the destination. The two main protocols at this layer are TCP (Transmission Control Protocol) which ensures reliable delivery and UDP (User Datagram Protocol) which is faster but less reliable.
  3. Internet layer: is responsible for addressing and routing data packets from the source to the destination across multiple networks. It uses IP addresses to identify devices and determine the best path for data to travel. The Internet Protocol (IP) operates at this layer.
  4. Network Access layer (Link layer): it is responsible for the physical transmission of data between devices on the same network. It handles how data is sent over physical hardware such as ethernet cables and network interface cards.

- TCP ensures reliable delivery by breaking data into numbered packets, sending them, reassembling them in order.

- IP handles addressing by assigning unique numeric addresses so data packets reach the right machine.
